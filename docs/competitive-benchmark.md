# 競合ベンチマーク

        ## 比較対象

        - Forest: 採用基準は入力負荷、記録粒度、証跡性、ローカル完結性。
- Focus To-Do: 採用基準は入力負荷、記録粒度、証跡性、ローカル完結性。
- Google Timer: 採用基準は入力負荷、記録粒度、証跡性、ローカル完結性。
- 学習管理アプリ: 採用基準は入力負荷、記録粒度、証跡性、ローカル完結性。

        ## 採用する基準

        - 入力負荷が低い。
        - 次アクションが明確。
        - ローカルまたは閉域で検証できる。
        - docs と release evidence から状態を追える。

        ## 差別化

        模試セクション、制限時間、休憩、採点メモを一連の実行フローにするに絞り、汎用ノートや大型サービスよりも閉域アルファの検証可能性を優先します。
